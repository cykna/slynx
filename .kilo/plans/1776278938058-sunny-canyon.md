# Plano: Filtragem de Instruções no Label

## Problema

Atualmente, o `Label` armazena instruções como um único `IRPointer<Instruction>` que aponta para um array contínuo:

```rust
// current
instruction: IRPointer<Instruction>  // ptr + length
```

Isso significa que todas as instruções são inseridas sequencialmente e não há como "filtrar" ou remover instruções intermediárias. Exemplo:

```rust
let value = insert(Raw(a));      // cria instruction[0] = a + b
insert(Return(value));           // cria instruction[1] = return a + b
// resultado: [instruction[0], instruction[1]]
// desejado:  [instruction[1]]  (apenas return)
```

## Solução Proposta

Mudar de `IRPointer<Instruction>` para um **array de ponteiros** (`SmallVec<[IRPointer<Instruction, 1>; N]>`):

```rust
// proposto
instructions: SmallVec<[IRPointer<Instruction, 1>; 4]>
```

### Benefícios DOD

1. **Filtragem explícita** - cada instrução é um ponteiro independente, pode-se remover/adicionar elementos específicos
2. **Controle de ordem** - instruções podem ser reordenadas ou removidas
3. **Metadados por instrução** - possível adicionar dados extras por instrução no futuro

### Mudanças Necessárias

1. **`middleend/src/ir/model/label.rs`**
   - Mudar `instruction: IRPointer<Instruction>` para `instructions: SmallVec<[IRPointer<Instruction, 1>; 4]>`
   - Atualizar métodos: `instruction()`, `insert_instruction()`, etc.

2. **`middleend/src/ir/instructions.rs`**
   - Atualizar `insert_instruction` para adicionar ao array ao invés de usar `increase_length`

3. **`middleend/src/ir/api.rs`**
   - Ajustar `get_label_instructions` para iterar sobre o array de ponteiros

### API Proposta

```rust
impl Label {
    // retorna array de ponteiros
    pub fn instructions(&self) -> &[IRPointer<Instruction, 1>]
    
    // adiciona nova instrução ao array
    pub fn insert_instruction(&mut self, ptr: IRPointer<Instruction, 1>)
    
    // remove instrução por índice (para filtragem)
    pub fn remove_instruction(&mut self, index: usize)
}
```

### Exemplo de Uso para Filtragem

```rust
// após gerar todas instruções, filtrar para manter apenas returns
let label = get_label(entry);
let instrs = label.instructions();
for (i, ptr) in instrs.iter().enumerate() {
    if !is_return_instruction(*ptr) {
        label.remove_instruction(i);
    }
}
```

## Decisão Pendente

Você prefere implementar a filtragem via:
1. **Remoção explícita** - após inserir, usar método para remover instruções específicas
2. **Inserção seletiva** - mudar a lógica de inserção para só adicionar quando necessário
3. **Outro abordagem** - tem outra ideia?
