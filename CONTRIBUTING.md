# Contributing to Slynx

## Welcome!

Thank you for considering contributing to Slynx! This document explains how you can help make this project better.

## How Can I Help?

### For Beginners
- Fix typos in documentation
- Add code examples
- Improve error messages
- Write tests

### For Developers
- Implement features from the Roadmap
- Optimize the compiler
- Add new backends
- Improve the type system

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/slynx`
3. Create a branch: `git checkout -b feature/my-contribution`
4. Make your changes
5. Run tests: `cargo test`
6. Run formatter: `cargo fmt`
7. Commit: `git commit -m "feat: clear description"`
8. Push: `git push origin feature/my-contribution`
9. Open a Pull Request

## Code Conventions

- Use `rustfmt` before committing: `cargo fmt`
- Follow Rust conventions
- Add tests for new features
- Document public functions with doc comments
- Keep commits atomic and well-described

## Reporting Bugs

Use GitHub Issues with:
- Clear description of the problem
- Code to reproduce the issue
- Expected vs actual behavior
- Environment (OS, Rust version)

## Suggesting Features

Open an Issue with:
- Feature description
- Use cases
- Examples of how it would be used

## Running Tests
```bash
cargo test
```

## Building Documentation
```bash
cargo doc --open
```

## Questions?

Feel free to open an issue for any questions or join our discussions!

## Code of Conduct

Be respectful, inclusive, and constructive in all interactions.
